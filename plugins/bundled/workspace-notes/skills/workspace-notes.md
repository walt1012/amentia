# Workspace Notes Skill

Use workspace notes only when the user asks Amentia to remember a durable project
preference, decision, convention, or follow-up.

- Capture stable facts, not transient chat phrasing.
- Keep each note short enough to be useful in future small-model context.
- Prefer project-specific wording over broad personal assumptions.
- Include why the note matters when that helps future work.
- Do not store secrets, tokens, credentials, or private account data.
- If a note is uncertain, ask before saving it.

Good notes preserve working context without turning memory into a transcript.
